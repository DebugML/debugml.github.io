The problem isn't just individual mistakes—it's how they cascade. Step 3 contains an <span style='color:orange; font-weight:bold'>**ungrounded error**</span> (incorrect information from context). Step 4 shows an <span style='color:red; font-weight:bold'>**invalid derivation**</span> (mathematical miscalculation). Step 5 demonstrates <span style='color:#880088; font-weight:bold'>**error propagation**</span>: while the logic is internally consistent with step 4, it's wrong because of the earlier error.

Current detection methods struggle because they evaluate entire chains simultaneously, getting confused by these propagated errors. An LLM judge might incorrectly mark step 5 as sound because it logically follows from step 4, missing that step 4 itself was based on faulty premises.


## Detecting Errors via Autoregressive Reasoning Entailment Stability (ARES)

To address this issue, we introduce Autoregressive Reasoning Entailment Stability (ARES), a probabilistic framework inspired by human reasoning, which assesses each reasoning step inductively.

It determines a step's soundness by calculating its entailment probability based _only_ on the set of preceding, sound claims, as shown in the figure below.

{% include gallery id="gallery_algo" layout="" caption="**Autoregressive Reasoning Entailment Stability (ARES)** When we verify a reasoning chain, we can break it down into base claims and derived claims. ARES checks each derived claim step-by-step, using previously-verified claims as premise at a probability." %}

A reasoning chain can be considered as "hard-sound" if every derived claim is perfectly entailed by all preceding claims.

**Definition. [Hard Soundness]** A claim in a reasoning chain is **hard sound** if it is entailed by all preceding claims.  
{: .notice--danger}

And a reasoning chain is hard sound if all claims in it are hard sound.

**Definition. [Hard Soundness (Chain)]** A reasoning chain is **hard sound** if all derived claims in the chain are entailed by preceding claims.  
{: .notice--danger}

However, this rigid definition is brittle, as a single minor error invalidates the entire chain.
Real-world LLM outputs rarely meet this standard and contain uncertainty.
We thus introduce soft soundness, "Autoregressive Reasoning Entailment Stability", to measure _how often_ each claim is entailed given previous sound claims, rather than whether it is _always_ entailed.
This is motivated by [soft stability](https://debugml.github.io/soft-stability/) and [hard stability](https://debugml.github.io/multiplicative-smoothing/) introduced for feature attributions in previous blog posts.

**Definition. [Soft Soundness]** A claim is **soft sound** if it is entailed on average across all sound combinations of preceding claims. Its **reasoning stability rate** \(\tau_k\) quantifies this probability for the \(k\)-th claim.  
{: .notice--info}

The probabilistic soundness has the following benefits:
1. **Fuzzy entailment:** In real life, some times the entailment of a claim is not completely black-and-white. Using a probabilistic entailment model allows accounting that uncertainty.
2. **Error Propagation Tolerance:** By averaging the entailment rates over all subsets of preceding claims that are sound, we allow robust estimations of entailment of a claim even in the presence of preceding errors.

### Certifying probabilistic soundness via efficient sampling

If we enumerate all probabilities of soundness of all combinations of claims, it soon becomes intractable.
If there are $m$ derived claims in a reasoning chain, then we need to check $\mathcal{O}(2^m)$ many perturbations!

Fortunately, previous works described in [this blog post](https://debugml.github.io/soft-stability/) proposed an efficient algorithm to estimate soft stability at a high accuracy via sampling.
We then adapt this algorithm to inductively certify soundness for each claim in a reasoning chain.

    <strong>Step 2.</strong> For each derived claim $k = 1, \ldots, m$:<br>

- **_Robust:_** previous errors do not adversely affect the current step.
- **_Causal:_** downstream steps do not affect the current step.

## Conclusion
In this blog post, we showed that ARES offers a novel approach to inductively assess reasoning soundness by probabilistically considering only previous sound claims as premises.
It is sample efficient and provides a more principled and reliable method for detecting errors in reasoning chains.
