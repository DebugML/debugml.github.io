function hexOrRgbaToRgba(c, alpha) {
  if (/rgba?\(/i.test(c)) {
    const nums = c.match(/[\d.]+/g).map(Number);
    const [r,g,b] = nums;
    return `rgba(${r},${g},${b},${alpha})`;
  }
  const m = c.replace('#','');
  const hex = (m.length === 3) ? m.split('').map(ch => ch+ch).join('') : m.padStart(6, '0');
  const val = parseInt(hex, 16);
  const r = (val >> 16) & 255, g = (val >> 8) & 255, b = val & 255;
  return `rgba(${r},${g},${b},${alpha})`;
}

A reasoning step could contain <span style='color:orange; font-weight:bold'>**ungrounded error**</span> that is incorrectly copying information from the context as in Step 3, <span style='color:red; font-weight:bold'>**invalid derivation**</span> mathematically or logically as in Step 4, or <span style='color:#880088; font-weight:bold'>**error propagation**</span> from previous mistakes as in Step 5.

Current error detection methods, such as LLM judges and Process Reward Models (PRMs), typically attempt to identify all errors at once.
However, they often fail to detect propagated errors when distracted by invalid premises.

In this work, we introduce **Autoregressive Reasoning Entailment Stability (ARES)**, a probabilistic framework inspired by human reasoning, which assesses each reasoning step inductively.
It determines a step's soundness by calculating its entailment probability based _only_ on the set of preceding, sound claims, as shown in the figure below.
We certify ARES with an efficient sampling algorithm that avoids exponential growth in cost as the number of claims increases.

Empirically, we find that our method accurately certifies both sound and unsound reasoning steps, particularly exceling in long chains prone to error propagation.

## Detecting Errors via Autoregressive Reasoning Entailment Stability (ARES)

ARES verifies a reasoning chain by assessing each step’s logical consistency with its preceding claims. In other words, it determines whether each derived claim can be justified from what has already been established as sound.

A reasoning chain can be considered as “hard-sound” if every derived claim is perfectly entailed by all preceding claims.

**Definition. [Hard Soundness]**  
A reasoning chain is **hard sound** if all derived claims in the chain are entailed by preceding claims.  
{: .notice--danger}

However, this definition is too rigid. We do not want to invalidate the entire reasoning chain because of a single early error, as LLMs can often self-correct in later steps.  

The figure below illustrates how ARES verifies reasoning chains by checking each derived claim against previously verified ones. In this **binary case**, a step is either judged sound (✓) or unsound (✗) depending on whether it follows logically from the preceding, sound claims.

{% include gallery id="gallery_algo" layout="" caption="**Autoregressive Reasoning Entailment Stability (ARES).** When verifying a reasoning chain, we can break it down into base claims and derived claims. ARES checks each derived claim step-by-step using previously verified claims as premises. This figure shows the binary case; later we generalize it to probabilistic entailment." %}

However, even this binary version remains overly discrete. It does not capture uncertainty in borderline entailment cases, where a claim may be only _partially supported_ by its premises. Moreover, real-world LLMs rarely meet this strict binary criterion due to linguistic ambiguity.

Fortunately, our [previous work](https://debugml.github.io/soft-stability/) provides the right vocabulary to express this concept probabilistically. The key idea is to evaluate the soundness of each reasoning step based on its **expected entailment probability**, conditioned only on preceding sound claims. This approach is inspired by how [humans](https://wires.onlinelibrary.wiley.com/doi/full/10.1002/wcs.20) tend to ignore dubious statements when reasoning, and by entailment strategies found in [fuzzy logic](http://www.scholarpedia.org/article/Fuzzy_logic).  

This culminates in the definition of **soft soundness** below.

**Definition. [Soft Soundness]**  
A claim is **soft sound** if it is entailed on average across all sound combinations of preceding claims. Its **reasoning stability rate** $\tau_k$ quantifies this probability for the $k$-th claim.  
{: .notice--info}

The probabilistic formulation of soundness offers two main benefits:
1. **Fuzzy entailment:** In real reasoning, entailment is not always black-and-white. A probabilistic model naturally accounts for this uncertainty.  
2. **Error propagation tolerance:** By averaging entailment rates over all subsets of preceding sound claims, we can robustly estimate each claim’s soundness even when some earlier steps are erroneous.




### Certifying probabilistic soundness via efficient sampling

If we enumerate all probabilities of soundness of all combinations of claims, it soon becomes intractable.
If there are $m$ derived claims in a reasoning chain, then we need to check $\mathcal{O}(2^m)$ many perturbations!

Fortunately, our [previous work](https://debugml.github.io/soft-stability/) addresses exactly this problem with an efficient algorithm to estimate soft stability at a high accuracy via sampling.
We then adapt this algorithm to inductively certify soundness for each claim in a reasoning chain.

<div class="notice--success">
  <strong>Algorithm. Autoregressive Reasoning Entailment Stability (ARES) Estimation</strong><br>
  Estimates the reasoning stability rate&nbsp;$\tau_k$ for each derived claim in a reasoning chain.<br><br>

  <div style="margin-left: 16px;">
    <strong>Step 1.</strong> Sample $N$ subsets of the $n$ base claims (with replacement), 
    where each base claim $C_i$ is included independently with prior probability&nbsp;$p_i$.<br><br>

    <strong>Step 2.</strong> For each derived claim $k = 1, \ldots, m$:<br>
    <div style="margin-left: 20px;">
      (a) For each sample, compute the entailment probability&nbsp;$p_{n+k}$ of the $k$-th claim 
      given the currently included claims, and estimate&nbsp;$\hat{\tau}_k = \frac{1}{N}\sum_j p_{n+k}^{(j)}$.<br>
      (b) Include the $k$-th claim in each sample for subsequent steps with probability&nbsp;$p_{n+k}$ 
      (Bernoulli sampling).<br>
    </div>
  </div>

  <br>
  <div style="margin-left: 16px;">
    <strong>Guarantee.</strong>  
    If $N \ge \frac{\log(2/\delta)}{2\varepsilon^2}$, then with probability at least $1 - \delta$,  
    the estimation error satisfies&nbsp;$|\hat{\tau}_r - \tau_r| \le \varepsilon$ for any claim&nbsp;$r$.
  </div>
</div>

### ARES detects more errors on diverse benchmarks

We also find that ARES performs well on 2 natural and 2 synthetic benchmarks:

<div id="benchmarks_gpt-4o-mini"></div>


<figcaption>
  <strong>(Benchmarks) GPT-4o-mini.</strong> ARES detects the most errors on 2 natural (PRMBench, DeltaBench) and 2 synthetic
  (ClaimTrees, CaptainCook) benchmarks. For synthetic sets we construct reasoning chains from ground-truth logic/recipe
  graphs and remove certain rules to induce propagated errors.
</figcaption>


We attribute this success to ARES being the only method that satisfies all key desiderata for error detection: 
- **_Robust:_** previous errors do not adversely affect the current step.
- **_Causal:_** downstream steps do not affect the current step.
- **_Sufficient:_** all relevant claims are included as premise when assessing.

## Conclusion
In this blog post, we showed that ARES offers a novel approach to inductively assess reasoning soundness by probabilistically considering only previous sound claims as premises.
It is sample efficient and provides a more principled and reliable method for detecting errors in reasoning chains.


For more details in thoeretical proofs and quantitative experiments, see our [paper](https://arxiv.org/abs/2507.12948) and [code](https://github.com/fallcat/ares).
